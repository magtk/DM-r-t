\section{0. Executive Summary}\label{executive-summary}

\subsection{Report Summary}\label{report-summary}

This report presents a complete Data Mining workflow based on the
CRISP-DM methodology. The analysis uses the penguin\_mess dataset, a
modified version of the Palmer Penguins dataset prepared for educational
purposes.

The primary objective of the project is to predict penguin species using
morphological measurements and selected categorical attributes.

The analysis follows the CRISP-DM (Cross-Industry Standard Process for
Data Mining) methodology, progressing through business understanding,
data understanding, data preparation, exploratory analysis, modeling,
and evaluation.

Two classification models were developed and compared. The results
demonstrate the importance of proper data preparation and feature
selection when building predictive models.

\subsection{Success Criteria}\label{success-criteria}

\end{itemize}

\section{1. Business Understanding}\label{business-understanding}

The objective of this project is to develop a predictive model capable
of identifying penguin species based on available measurements and
characteristics.

Although the dataset is educational in nature, the problem reflects a
common Data Mining scenario in which historical observations are used to
classify new cases. The project also aims to demonstrate the complete
CRISP-DM process rather than maximize predictive performance.

The project aims to answer the following question:

\begin{quote}
Can penguin species be accurately predicted using the available
measurements and characteristics?
\end{quote}

\subsection{Project Scope}\label{project-scope}

The analysis includes: - data understanding and quality assessment; -
data preparation and cleaning; - exploratory data analysis (EDA); -
development of classification models; - model evaluation and comparison;
- interpretation of results.

The project does not aim to build a production-ready system. Instead, it
serves as a demonstration of the CRISP-DM process and fundamental Data
Mining techniques.

\subsection{Problem Statement}\label{problem-statement}

Given a set of penguin observations containing morphological
measurements and categorical characteristics, the objective is to
develop a predictive model capable of correctly identifying the species
of each penguin.

The project investigates which variables are most informative for
species classification and evaluates whether reliable predictions can be
obtained from the available data.

\subsection{Objective}\label{objective}

The objective of this project is to investigate whether penguin species
can be accurately predicted using available morphological measurements
and categorical attributes.

To achieve this goal, the dataset will be explored, cleaned, and
prepared for modeling. Multiple classification models will be developed
and compared in order to identify the most suitable approach for this
prediction task.

\subsection{Expected Outcome}\label{expected-outcome}

The expected outcome of this project is the development of a
classification model capable of predicting penguin species based on
available measurements and categorical attributes. It is expected that
the analysis will identify variables that are most informative for
species classification and reveal patterns that distinguish different
species.

Furthermore, the project is expected to demonstrate the impact of data
preparation and feature selection on model performance, as well as
provide a comparison of alternative modeling approaches.

\subsection{Data Quality Assessment}\label{data-quality-assessment}

The dataset contains several missing values affecting both numerical and
categorical variables. Missing values are particularly common in the
\texttt{notes} attribute, where the majority of observations contain no
information.

In addition, the dataset includes an identifier variable (\texttt{id})
which is not expected to contribute to predictive performance and may
require removal before modeling.

The \texttt{notes} variable contains missing values in the majority of
observations and provides limited analytical value. Therefore, it was
excluded from further analysis.

The correlation matrix provides a quantitative summary of linear
relationships between numerical variables. Some variables exhibit
stronger associations than others, suggesting that they may contain
related information. Correlation analysis can help identify potentially
redundant features and support feature selection during model
development. However, correlation measures only linear relationships and
should always be interpreted alongside exploratory visualisations and
domain knowledge.

ℹ️️ \textbf{Correlation does not imply dependence}

Correlation measures only linear relationships between variables. A low
correlation coefficient does not necessarily indicate the absence of a
relationship.

Variables may exhibit non-linear dependencies that are not captured by
correlation analysis. A classic example is the \emph{Anscombe Quartet},
where several datasets share nearly identical summary statistics and
correlation coefficients while displaying very different patterns when
visualized.

For this reason, correlation analysis should always be complemented by
exploratory visualisations and domain knowledge.

\subsection{Discussion}\label{discussion}

Two classification models were developed and evaluated: a Decision Tree
and a Random Forest.

The Decision Tree model offers a high level of interpretability. The
resulting tree structure can be visualized and easily understood, making
it useful for explaining classification decisions. However, a single
tree may be sensitive to variations in the training data and can be
prone to overfitting.

The Random Forest model addresses some of these limitations by combining
multiple decision trees into an ensemble. This approach generally
improves predictive performance and robustness while reducing the impact
of individual tree errors. The trade-off is reduced interpretability, as
the model's decision-making process becomes more complex.

Overall, both models demonstrate how machine learning techniques can be
applied to classification problems. The comparison highlights the common
trade-off between model interpretability and predictive performance.

\subsection{Limitations}\label{limitations}

Several limitations should be considered when interpreting the results.

First, the dataset required extensive cleaning due to missing values,
inconsistent categorical labels, duplicate records, and implausible
observations. The quality of the final models therefore depends on the
assumptions made during the data preparation stage.

Second, the analysis was conducted using a single train-test split.
Different random splits or cross-validation procedures could produce
slightly different performance estimates.

Third, the project focused on a limited number of classification
